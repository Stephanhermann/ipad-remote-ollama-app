# Privacy Policy

## English

`ipad-remote-ollama-app` stores chat sessions, server profiles, and local app settings on the user's device.

The app does not require a mandatory cloud account and does not send data to the developer by default.

Network traffic is only sent to servers explicitly configured by the user, such as a remote Ollama endpoint or a user-managed reverse proxy.

Depending on how the user configures the app, the following data may be processed:

- Chat messages entered by the user
- Model responses returned by the configured server
- Server URLs and authentication settings entered by the user
- Images selected by the user for multimodal prompts
- Speech input processed through Apple system frameworks on the device

The developer does not guarantee, control, or monitor the privacy practices of third-party or self-hosted servers configured by the user.

Users are responsible for securing their own Ollama infrastructure, especially when exposing it over a network.

If the app is distributed publicly, this policy should be reviewed again before release.

## Deutsch

`ipad-remote-ollama-app` speichert Chat-Verlaeufe, Serverprofile und lokale App-Einstellungen auf dem Geraet des Nutzers.

Die App benoetigt kein verpflichtendes Cloud-Konto und uebermittelt standardmaessig keine Daten an den Entwickler.

Netzwerkverkehr wird nur an Server gesendet, die der Nutzer selbst konfiguriert, zum Beispiel an eine entfernte Ollama-Instanz oder an einen selbst verwalteten Reverse-Proxy.

Je nach Nutzung und Konfiguration koennen insbesondere folgende Daten verarbeitet werden:

- vom Nutzer eingegebene Chat-Nachrichten
- Modellantworten des konfigurierten Servers
- vom Nutzer eingetragene Server-URLs und Authentifizierungsdaten
- vom Nutzer ausgewaehlte Bilder fuer multimodale Prompts
- Spracheingaben, die ueber Apple-Systemframeworks auf dem Geraet verarbeitet werden

Der Entwickler kontrolliert oder ueberwacht keine Datenschutzpraxis von Servern Dritter oder selbst gehosteten Instanzen, die vom Nutzer eingetragen werden.

Der Nutzer ist selbst dafuer verantwortlich, seine Ollama-Infrastruktur sicher zu betreiben, insbesondere bei einer Freigabe ueber das Netzwerk.

Vor einer oeffentlichen Veroeffentlichung sollte diese Datenschutzerklaerung noch einmal rechtlich und inhaltlich geprueft werden.
